Some standard method in a view

Create:

  • Constructor: Called for creating view in code OR inflating from XML. We can have different types of constructors defined.
  • onFinishInflate: Called when view and all of the children has been inflated from XML file.

Layout:

  • onMeasure: Called to determine the size requirement of a view and it’s children.
  • onLayout: Called when view should assign a position and size to all its children.
  • onSizeChange: Called when the size of the view has changed.

Drawing:

  • onDraw: Called when the view should render its content.

If you want to implement a custom view (say a chevron), you can start to just override the onDraw method.

Position Coordinator

Android view coordinator origin point is on the top left, as shown in the below.

We can use getLeft(), getTop(), getBottom(), and getRight() to know the coordinate of the view so that we know its position. Actually getLeft() and getTop() are the most useful ones, and getBottom() and getRight() are just convenient values that can be calculated.
